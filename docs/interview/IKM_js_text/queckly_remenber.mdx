题目：以下哪些关于 JavaScript 中上下文的陈述是正确的？

A. JavaScript 中的上下文与作用域相同。

B. 上下文可以通过代码中的关键字 this 来访问。

C. 函数执行的上下文可以是可变的。

D. 全局函数（如 parseInt()）的上下文是 null。

E. 上下文指的是代码执行时所在的 JavaScript 对象。

解释：

A. The context in JavaScript is the same as scope.

错误。在 JavaScript 中，上下文（context）和作用域（scope）是两个不同的概念。作用域决定了变量和函数在代码中的可见性和可访问性，而上下文（或称为 this 的值）则是指函数执行时所在的对象环境。
B. The context can be accessed inside the code by the keyword this.

正确。在 JavaScript 中，this 关键字用于访问函数的上下文（context）。它指的是函数执行时所在的对象。如果函数是全局对象（如浏览器中的 window）的方法，那么 this 就指向全局对象。
C. The context under which a function is executed can be variable.

正确。JavaScript 中函数的上下文（即 this 的值）是可变的，它取决于函数是如何被调用的。例如，通过对象方法调用时，this 指向该对象；作为普通函数调用时，this 指向全局对象（在严格模式下为 undefined）；使用 call、apply 或 bind 方法时，可以显式设置 this 的值。
D. The context of global functions (such as parseInt() ) is null.

错误。全局函数（如 parseInt()）的上下文（this）在非严格模式下是全局对象（在浏览器中为 window），而不是 null。在严格模式下，全局函数的 this 值是 undefined。
E. The context refers to the JavaScript object under which the code is executed.

正确。这个选项正确地描述了上下文的概念，即上下文是指代码执行时所在的 JavaScript 对象。这个对象决定了 this 的值，以及在该上下文中可以访问的属性和方法。

B. 正确。闭包是函数和函数创建时词法环境（即作用域）的组合。这意味着闭包不仅包含了函数本身，还包含了函数被创建时作用域内的所有变量和状态。

关于 JavaScript 中的 Map 和 WeakMap，以下哪些陈述是正确的？

A. 可以很容易地获取 Map 的大小，但 WeakMap 的大小必须手动记录。

B. WeakMap 是 Map 的简单实现，用于小型和短生命周期的映射。

C. Map 对象的键是可枚举的，而 WeakMap 的键不是。

D. WeakMap 受垃圾回收机制影响，但 Map 不受影响。

E. WeakMap 可以持有字符串值，而 Map 可以持有任何类型的值。

WeakMap 和 Map 都是 JavaScript 中的对象，它们本身都是垃圾回收的目标。但是，它们对键的处理方式不同：WeakMap 对键进行弱引用，允许垃圾回收机制回收那些不再被其他引用所持有的键所指向的对象；而 Map 则对键进行强引用，即使键所指向的对象不再被其他引用所持有，这些对象也不会被垃圾回收机制回收，直到 Map 本身被销毁或键被显式地从 Map 中删除。

WeakMap 和 Map 都可以持有任何类型的值，包括字符串、数字、对象等。但是，它们的键类型有所不同：WeakMap 的键必须是对象（在 ES6 规范中，原始数据类型如字符串或数字会被隐式地封装为对象），而 Map 的键可以是任何类型，包括原始数据类型和对象。

A. Names are case sensitive.（名称是区分大小写的。）
