### XSS 攻击 防御 dompurify

```shell
# 安装 dompurify
npm install dompurify
```

```js
import DOMPurify from "dompurify";

var clean = DOMPurify.sanitize(dirty);

let text = "<img/src=1 onerror=alert(1)>"; // 用户填写的可能有风险的内容

// 没有防护的时候，这里其他用户浏览器中会意外执行某一用户填写风险脚本
return (
  <div
    dangerouslySetInnerHTML={{ __html: text.replace(/\n/g, "<br />") }} // 将换行符替换成br，为了显示换行
  />
);

// 用法。这里不会意外执行某一用户填写风险脚本
return (
  <div
    dangerouslySetInnerHTML={{
      __html: DOMPurify.sanitize(text.replace(/\n/g, "<br />")),
    }}
  />
);
```

### cross-env

process.env.NODE_ENV 能拿到当前环境变量里中的 NODE_ENV 变量，可以通过配置 package.json 中的 script 命令配置临时环境变量，

例如

```json
{
  "scripts": {
    "build:test": "cross-env NODE_ENV=test umi build ",
    "build:prod": "cross-env NODE_ENV=prod umi build ",
    "build:prodbak": "cross-env NODE_ENV=prodbak umi build "
  }
}
```

windows不支持NODE_ENV=development的这样的设置方式。可以使用cross-env命令来实现全开发平台的兼容。
要使用该命令的话，需要在项目中安装该命令：

npm 安装方式
```shell
npm install --save-dev cross-env
```
